# Android Build Setup

This document provides instructions for setting up the Android build environment for CryptoMafia.

## Issue Resolution

**Problem**: `tauri android build` fails with "Android Studio project directory doesn't exist. Please run tauri android init and try again."

**Root Cause**: The `tauri android init` command requires a properly configured Android development environment with SDK and NDK.

## Prerequisites

To build the Android APK, you need:

1. **Android SDK and NDK**
2. **Java 17**
3. **Rust with Android targets**

## Environment Setup

### 1. Install Android SDK and NDK

You can either:

**Option A: Use Android Studio**
- Install Android Studio
- Install SDK and NDK through the SDK Manager
- Set `NDK_HOME` environment variable

**Option B: Use Command Line Tools**
```bash
# Download Android command line tools
wget https://dl.google.com/android/repository/commandlinetools-linux-11076708_latest.zip
unzip commandlinetools-linux-11076708_latest.zip -d android-sdk

# Install SDK and NDK
export ANDROID_SDK_ROOT=~/android-sdk
export PATH=$ANDROID_SDK_ROOT/cmdline-tools/latest/bin:$PATH
sdkmanager "platform-tools" "platforms;android-34" "ndk;26.1.10909125"

# Set NDK_HOME
export NDK_HOME=$ANDROID_SDK_ROOT/ndk/26.1.10909125
```

### 2. Install Java 17

```bash
# Ubuntu/Debian
sudo apt-get install openjdk-17-jdk

# macOS
brew install openjdk@17
```

### 3. Install Rust Android Target

```bash
rustup target add aarch64-linux-android
```

### 4. Install Tauri CLI

```bash
npm install -g @tauri-apps/cli
# or
cargo install tauri-cli
```

## Building the APK

Once the environment is set up:

```bash
cd src-tauri

# Initialize Android project (this creates the gen/android directory)
tauri android init

# Build the APK
tauri android build --apk --target aarch64
```

The APK will be generated in: `src-tauri/gen/android/app/build/outputs/apk/`

**Important**: The `gen/android` directory is generated by `tauri android init` and should not be committed to version control. It's included in `.gitignore`.

## CI/CD

The GitHub Actions workflow automatically handles the Android build setup. See `.github/workflows/build-android-app.yml` for the complete CI configuration.

## Troubleshooting

### Error: "Android Studio project directory doesn't exist"

This error occurs when `tauri android init` hasn't been run successfully. To fix:

1. Ensure Android SDK and NDK are properly installed and `NDK_HOME` is set
2. Run `tauri android init` before building
3. If the command reports success but doesn't create files, check your Android environment setup

**The fix**: Always run `tauri android init` before `tauri android build` in an environment with proper Android SDK/NDK setup.

### Error: "NDK_HOME environment variable isn't set"

Set the NDK_HOME environment variable to point to your Android NDK installation:

```bash
export NDK_HOME=/path/to/android/ndk/26.1.10909125
```

### Error: Build failures

1. Ensure frontend is built: `cd frontend && npm run build`
2. Check that all Android targets are installed: `rustup target list --installed`
3. Verify Java version: `java -version` (should be 17)

## Project Structure

After running `tauri android init`, the Android project will include:

- `src-tauri/gen/android/AndroidManifest.xml` - App permissions and activity configuration
- `src-tauri/gen/android/MainActivity.java` - Main Tauri activity
- `src-tauri/gen/android/build.gradle` - Android build configuration
- Resource files (strings, styles, icons)
- Proper directory structure for Tauri Android integration

**Note**: These files are generated automatically and should not be committed to version control.
